# Инженерные решения

## Execution model

- Go backend предоставляет HTTP API и статический WebUI.
- Работа с Asterisk выполняется через SSH с ограниченными таймаутами.
- Произвольное выполнение команд не является нормальным API: используется
  allowlist команд и контролируемые пути конфигурации.
- Операции записи выполняются backup-first; после проверки синтаксиса следует
  узкий reload, затем post-check и acceptance.

## Safety model

- простой и инженерный режим уменьшают риск ошибочного действия;
- RBAC отделяет read-only диагностику от опасных операций;
- readiness и runtime-policy выполняются read-only;
- UI показывает источник истины и слой-владелец, чтобы оператор не исправлял
  симптом в неправильной конфигурации;
- rollback проектируется до применения изменения.

## Observability и эксплуатация

- рабочее место агрегирует состояние зависимых подсистем;
- оператор получает следующий шаг, а не только красный/зелёный индикатор;
- evidence и runbook доступны из того же интерфейса;
- метрики продукта могут экспортироваться в Prometheus;
- browser regression проверяется Playwright, backend — Go tests/vet/format.

## Границы showcase

Скриншоты подтверждают дизайн и реализованный интерфейс, но не являются
доказательством конкретной production-доступности, нагрузки или SLA. Все
операционные значения на изображениях синтетические.
