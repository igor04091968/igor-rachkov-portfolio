# Сценарий демонстрации на 7 минут

## 0:00–1:00 — рабочее место

Показать `01-workspace-overview.png`: один экран отвечает на вопросы «доступна
ли АТС», «готов ли production-контур», «соответствует ли runtime policy»,
«готовы ли DHCP/provisioning/notifier» и «какой следующий безопасный шаг».

## 1:00–2:00 — PJSIP

Показать `02-endpoints-and-trunks.png`: текущие регистрации отделены от Match,
trunk-и — от пользовательских аппаратов. Простой режим не перегружает оператора
техническими полями.

## 2:00–3:00 — очереди

Показать `03-queues-runtime-policy.png`: runtime `queue show` не объявляется
единственным источником истины. UI связывает состояние с policy registry и
помогает локализовать drift в правильном слое.

## 3:00–4:00 — карта системы

Показать `04-system-source-of-truth-map.png`: слои, источники истины, runtime
checks и правила для оператора/администратора оформлены как часть продукта.

## 4:00–5:00 — runbook

Показать `05-operator-runbook.png`: сначала blast radius и evidence, потом
изменение. Runbook встроен в интерфейс, поэтому первая линия не ищет актуальную
инструкцию по чатам и старым файлам.

## 5:00–6:00 — управляемое изменение

Показать `06-schedule-and-safe-change.png`: расписание — бизнес-функция, но её
применение проходит через backup, syntax validation, reload и проверку.

## 6:00–7:00 — итог

Показать мобильный экран и завершить: «Проект снижает количество ручных SSH- и
SQL-операций, но сохраняет эксплуатационные guardrails, RBAC и проверяемый
rollback».
