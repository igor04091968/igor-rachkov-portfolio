# Как представить портфолио на собеседовании

## 1. Профессиональный профиль — одна минута

«Я занимаюсь эксплуатацией и автоматизацией разнородной инфраструктуры: Linux,
Windows и Active Directory, Proxmox и виртуализация, VLAN, пакетная фильтрация,
несколько VPN-контуров, DNS, почта, телефония и мониторинг. Для автоматизации
использую Rust, Go, Bash, PowerShell и Ansible. Моя сильная сторона — связать
эти компоненты в проверяемый рабочий контур, а не обслуживать каждый из них
изолированно».

## 2. Основной SRE-кейс — три минуты

Открыть кейс AWatch-rus. Показать путь данных от Windows-коллектора до отчёта и
объяснить главную проблему: сервис может быть запущен, но данные уже устарели.
Рассказать о проверках свежести, многоуровневом health-check, отчётности,
безопасном восстановлении и Ansible-развёртывании.

## 3. Сеть и безопасность — две минуты

Перейти к pfSense/OpenVPN. Объяснить, почему пустой или огромный счётчик
блокировок одинаково бесполезен без классификации. Показать логику отделения
шума от вероятных штормов и атак, статистику по каждому VPN-клиенту и
календарные недельные/месячные сводки.

Затем коротко показать отдельный кейс Proxmox и сети: VM/LXC, storage и
snapshots, сетевые мосты и VLAN, маршрутизация, NAT, пакетные фильтры и несколько
VPN-профилей с разными маршрутами.

## 4. Автоматизация — две минуты

Коротко объяснить выбор инструментов: Rust для надёжных автономных CLI, Go для
backend, shell для небольших системных задач, PowerShell для Windows, Ansible
для повторяемого развёртывания. Сделать акцент на dry-run, таймаутах, атомарной
записи, блокировках и откате.

## 5. ИИ в ежедневной работе — две минуты

Показать, что ИИ используется не только для написания текста. Рассказать о
семантическом поиске по документации и логам, работе Codex с локальным кодом,
быстрых проходах Pollinations, дополнительном ревью через Copilot/OpenCode,
MCP-интеграциях и browser automation. Завершить рабочим циклом: контекст,
гипотеза, diff, профильный тест, применение и проверка результата.

## 6. Корпоративные сообщения и телефония — две минуты

Показать связку XMPP/Jabber, ejabberd, Miranda IM и Asterisk. Рассказать об
уведомлениях для очередей и звонке по клику: Miranda отправляет XMPP-запрос,
Asterisk сопоставляет JID с внутренним номером, вызывает телефон инициатора и
соединяет его с выбранным абонентом. Показать также запуск конференции из
сообщения. Этот кейс хорошо демонстрирует интеграцию нескольких давно работающих
систем без замены привычного рабочего клиента.

## 7. Asterisk — только как дополнительный пример

Открывать интерфейс Asterisk, если разговор перешёл к телефонии, разработке
внутренних инструментов или руководству первой линией. Показать один экран и
объяснить разделение ролей, встроенный runbook и безопасное применение
изменений. Не проводить экскурсию по всем кнопкам без запроса собеседника.

## Завершение

«Эти проекты разные по предметной области, но мой подход один: сначала сделать
состояние наблюдаемым, затем автоматизировать повторяемые действия и обязательно
оставить проверку результата и путь возврата».
